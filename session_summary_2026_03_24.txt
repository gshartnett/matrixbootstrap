2. THREE BUGS FIXED IN BootstrapSystemComplex (committed to feat branch):

   Bug 1 — coo_matrix not subscriptable (bootstrap.py load_config_cache):
     load_npz returns coo_matrix; solver's gradient path does matrix[i] (row
     indexing) which requires CSR. Fix: .tocsr() instead of .copy() when loading
     quadratic constraints from cache.

   Bug 2 — Wrong real-basis projection for complex operator coefficients
     (bootstrap_complex.py single_trace_to_coefficient_vector):
     For operator S = sum_j z_j O_j, the correct real-basis coefficient vector
     (projecting onto Re(<tr(S)>)) is [Re(z), -Im(z)] @ K, NOT [z, z] @ K.
     The wrong formula gave complex constraint vectors for operators with imaginary
     coefficients (e.g. Myers term i*eps_ijk tr(X_i X_j X_k)), causing CLARABEL
     to reject the problem with "Array must be contiguous" or return infeasible.

   Bug 3 — Non-deterministic operator ordering (bootstrap_complex.py
     generate_operators_truncated):
     operator_list = list(set([...])) has random ordering between Python processes
     due to hash randomisation. The null space matrix K is cached with one ordering,
     then loaded in a new process with a different operator_dict ordering, making
     L @ K != 0 (corrupt null space). Fix: use sorted(set([...])).
     IMPORTANT: If you regenerate the per-config cache after this fix, it will be
     correct. Any cache built before this fix (with list(set())) is corrupt and must
     be deleted and rebuilt.

   Bug 4 — Wrong bootstrap class in config_utils:
     generate_config_bmn was using "BootstrapSystem" instead of
     "BootstrapSystemComplex". Fixed.

   Bug 5 — Feasibility threshold too tight (mini_bmn_feasibility_bound.py):
     TOL_QUAD=1e-6 caused false-negative infeasibility for high E_trial where
     Newton residual stabilises at ~1.5e-6 (within CLARABEL's 1e-5 precision).
     Fix: use EPS=1e-5 as the feasibility threshold.

3. NEW SCRIPT: scripts/mini_bmn_feasibility_bound.py
   Feasibility-based lower bounds on mini-BMN ground state energy.
   Uses binary search: fix <H>=E_trial, minimize ||x||, search for infeasibility.

== Mini-BMN Model ==

Model: MiniBMN(nu, lambda)
  g2 = nu^2, g3 = 3*nu*sqrt(lambda), g4 = lambda
  H = g2/2 * sum_i tr(X_i^2 + P_i^2)
    + i*g3/3 * eps_ijk tr(X_i X_j X_k)    ← Myers term, imaginary coeff!
    + g4/4 * commutator^2 terms
  3 matrices (X0,X1,X2), 3 momenta (Pi0,Pi1,Pi2)
  Requires BootstrapSystemComplex (not BootstrapSystemReal) due to cubic term.

Scale:
  L=2: bootstrap_matrix_dim=43, param_dim_null=12
       86x86 real PSD block (43x43 complex -> 2x2 block)
       Very fast (~seconds per SDP)
  L=3: bootstrap_matrix_dim=259, param_dim_null=??
       518x518 real PSD block
       ~6x larger matrix -> ~216x more work per SDP step
       Expect slow but potentially tractable with CLARABEL
